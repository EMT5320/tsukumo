# Hermes Instructions for tsukumo

This repository is managed by Trellis. Hermes should use this file as the project-level entry point and keep the Trellis context discoverable even when the working directory is a subdirectory.

## Trellis context

Before making code, workflow, or task changes, inspect the relevant Trellis files:

- `.trellis/workflow.md` — development phases, task routing, and workflow-state prompts.
- `.trellis/config.yaml` — project-level Trellis configuration.
- `.trellis/tasks/` — active and archived task context.
- `.trellis/spec/` — project conventions and coding guidelines.
- `.trellis/workspace/` — per-developer journals and session traces.
- `.agents/skills/` — reusable Trellis skills shared with compatible agents.
- `.codex/agents/` — optional Codex custom subagents.

## Trellis CLI

The Trellis CLI is expected to be available as `trellis` and `tl`. Use it when it is safer than manual file inspection or when operating Trellis-managed state:

- `trellis --version`
- `trellis mem help`
- `trellis channel --help`
- `trellis update --dry-run`

If the global Trellis CLI version differs from `.trellis/.version`, do not run mutating Trellis commands blindly. Check the mismatch first and prefer dry-run commands before applying updates.

## Editing rules

- Do not manually edit Trellis-managed blocks unless intentionally customizing local Trellis behavior.
- Preserve user changes in `.trellis/` and platform directories; read current files before patching.
- Keep Hermes-specific guidance here concise. Put broad cross-agent rules in `AGENTS.md`, and put project coding conventions in `.trellis/spec/`.
